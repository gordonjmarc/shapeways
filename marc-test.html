  • Q: Describe the differences between the IE6, IE7, and W3C box models.

    IE had what was known as the box model bug. Basically the width didn't include border and padding as well in it's calculation which was required for compliance to W3c. It required developers to use tricks to make markup look correctly in differnt versions of IE. IE has both standards and quirks mode, quirks mode allowed for compatibiltiy with older markup. This mode was influenced by the DOCTYPE tag at the begining of the markup... In IE 6 if there wasn't anything before the DOCTYPE it would force quirks mode, yet they eliminated this functionality in IE 7.

    Describe the pros and cons of version control.

    • Some Pros: Version control promotes incremental changes in code to be stored continuously during progress of the project. It leaves hooks or points in the code where previous versions may be retrieve, or compared to the current HEAD or latest revision. It allows developers to have a common place to both store/retrieve code. It promotes transparency and availability of latest
    • Some Cons: Source control requries best practices, and when under utilized can create quite a mess. The quality requires frequent commits, good use of forking/branching, meaningful commit messages and things like this. If it is simply used as a file server for source, It can make things more confusing and hinder development progress.
